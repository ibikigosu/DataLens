# Issue tracker: GitHub

Issues and PRDs for this repository live in GitHub Issues at `ibikigosu/DataLens`.

## Conventions

- Use GitHub Issues for plans, PRDs, implementation slices, bugs, and follow-up work.
- Use the `gh` CLI with the explicit repository argument `--repo ibikigosu/DataLens` until a local Git remote exists.
- Do not create commits, push branches, or open pull requests as a side effect of issue management.
- Preserve the teaching sequence by keeping implementation issues narrow, chronological, and independently reviewable.
- A feature should normally have its own issue, branch, commit sequence, and pull request.
- Do not close or modify parent planning issues when creating implementation issues.

## Skill behavior

When a skill says to publish to the issue tracker, create a GitHub issue in `ibikigosu/DataLens`.

When a skill says to fetch a ticket, read the full issue body, labels, and comments.
