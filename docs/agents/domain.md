# Domain documentation

DataLens uses a single-context documentation layout.

Before implementation or architecture work, read the following files when they exist:

- `CONTEXT.md` for domain language and project boundaries.
- `docs/adr/` for accepted architectural decisions.

Use DataLens domain terms consistently in issues, tests, documentation, and implementation.

If a proposed change conflicts with an accepted architecture decision, identify the conflict instead of silently replacing the decision.
