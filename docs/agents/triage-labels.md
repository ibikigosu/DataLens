# Triage labels

The engineering skills use canonical workflow roles internally.
Map those roles to ordinary GitHub labels as follows.

| Internal role | GitHub label | Meaning |
| --- | --- | --- |
| `needs-triage` | `planning` | The issue still needs review or refinement |
| `needs-info` | `question` | More information or a decision is required |
| `ready-for-agent` | `ready` | The issue is fully specified and can be implemented |
| `ready-for-human` | `help wanted` | The issue needs direct human attention or participation |
| `wontfix` | `wontfix` | The issue will not be implemented |

Do not expose internal agent-oriented vocabulary in public issue titles or descriptions.
